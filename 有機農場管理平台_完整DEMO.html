安全性升級
現有系統安全性問題診斷與修補方案
發現 4 項高風險安全性問題，建議在上線前優先修補，以保護農友個資與系統安全。
安全性問題清單
高
SQL 注入漏洞（SQL Injection）
login.php 直接將 $_COOKIE['key'] 嵌入 SQL 查詢，攻擊者可竊取或篡改全部資料庫。
修復方案：改用 PDO Prepared Statements
高
不安全的密碼雜湊（SHA1）
資料庫中密碼使用 SHA1，已可被彩虹表破解，應改用 bcrypt 或 Argon2。
修復方案：改用 password_hash() + PASSWORD_BCRYPT
高
資料庫帳密明文儲存
config.php 中直接寫死帳密，任何有原始碼存取權的人都能取得。
修復方案：改用 .env 環境變數，不納入版控
中
Session 安全設定不足
缺少 session_set_cookie_params() 中的 HttpOnly、Secure、SameSite 設定，易受 XSS 劫持。
修復方案：加入完整 session cookie 設定
中
使用廢棄的 mysql_* 函數
function.php 使用 mysql_connect()、mysql_query() 等已在 PHP 7 移除的函數，需升級至 PDO 或 mysqli。
修復方案：全面替換為 PDO 或 mysqli_*
低
PHP 版本過舊（5.2）
PHP 5.2 已於 2011 年停止維護，含有大量已知安全漏洞。
修復方案：升級至 PHP 8.2+
