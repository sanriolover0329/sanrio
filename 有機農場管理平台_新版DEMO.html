優化方案與未來擴充
基於現有原始碼分析所提出的改善建議
安全性問題（高優先）
1. SQL 注入風險：config.php 中多處查詢直接嵌入 $_COOKIE、$_SESSION 值，未使用 Prepared Statements。
2. 明文密碼儲存：資料庫中部分密碼使用 SHA1（已不安全），應改用 password_hash() + bcrypt。
3. 資料庫帳密外露：config.php 內含明文帳密，應改用環境變數（.env）管理。
4. Session 安全：ex_session.php 的 session 設定不完整，缺少 HttpOnly、SameSite 旗標。
效能優化建議
1. 資料庫引擎：表格使用 MyISAM，建議改用 InnoDB（支援交易、外鍵、更佳的並發）。
2. PHP 版本：原始碼基於 PHP 5.2，已停止更新。建議升級至 PHP 8.2+，效能提升 50%+。
3. jQuery 版本：使用 jQuery 1.4/1.7（2011年），建議升級至 jQuery 3.x 或改用原生 JS。
4. SQL 查詢優化：home.php 中多次 COUNT 查詢可合併，減少資料庫往返。
架構重構建議（中長期）
建議從「傳統 PHP 多頁式」逐步轉向「前後端分離」架構，提升可維護性與擴充性。
後端可保留 PHP（升級 Laravel），或改用 Node.js；前端改用 Vue 3 或 React。
